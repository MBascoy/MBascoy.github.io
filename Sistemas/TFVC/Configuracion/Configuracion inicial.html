Descripcion

Creacion y configuracion inicial de un proyecto con Team Foundation Version Control (TFVC).

NOTA: el sistema de control de versiones TFVC antes usaba el servidio Team Foundation Server (TFS), pero ahora el servicio se llama Azure DevOps Server.

Documentación TFVC Microsoft

Metodo

Para crear un proyecto con TFVC primero tenemos que crear un repositorio en Azure DevOps:

  • Nos vamos a la página de Azure DevOps
  • Seleccionamos la opción de New project.
  • Le damos un nombre a nuestro proyecto y le damos a create:
  • Ahora mismo ya tenemos nuestro proyecto creado, pero por defecto los proyectos utilizan el control de versiones de GIT, tenemos que crear un nuevo repositorio con TFVC, para ello nos vamos a Repos.
  • Dentro de repos podemos ver que hay creado un repositorio de GIT, le damos a New repository, para crear un repositorio TFVC.
  • En el tipo de repositorio seleccionamos TFVC, y le damos a crear:
  • Despues de realizar todos estos pasos ya podemos ver que nuestro proyecto tiene un repositorio TFVC asignado:
  • A continuación tenemos que asociar dicho repositorio a un Workspace en nuestro Visual Studio, iniciamos visual studio y selecionamos la opción Clone a repository:
  • Seleccionamos clonar desde el repositorio de Azure DevOps:
  • Seleccionamos nuestros repositorio:
  • Mapeamos el repositorio con una carpeta local, por defecto los repositorios se almacenan en: %USERPROFILE%\Source\Workspaces\
  • Por defecto visual studio crea un nuevo workspace para el repositorio:
  • Podemos modificar el nombre del workspace si queremos:
  • Si queremos tambien podemos mapear carpetas adicionales al repositorio:

Añadir solucion al repositorio

En este punto ya tenemos nuestro repositorio creado y mapeado con una carpeta local en nuestro Visual Studio, ahora crearemos una nueva solucion y la meteremos dentro del repositorio.

  • Desde el team explorer hacemos click en New:
  • Creamos un nuevo proyecto (y solucion), en este caso usaremos un programa de consola como ejemplo:
  • Una vez creado el proyecto (solucion) veremos que la lista de soluciones sigue vacia, tenemos que irnos al Solution explorer para añadirla:
  • Una vez en el Solution Explorer hacemos click derecho en la solución y le damos a Add Solution to Source Control:
  • Si volvemos al Team Explorer vemos que la solución ahora aparece en la lista:

    • NOTA: si tenemos varias soluciones la que está en negrita es la que está activa en el Solution Explorer, si ninguna de las soluciones está en negrita quiere decir que la solución activa está en otro Workspace

  • Ahora solo nos queda subir la nueva solución que hemos añadido al repositorio central, para esto nos vamos a Pending changes:
  • En esta pestaña simplemente tenemos que pulsar el boton check in para subir los cambios, en la parte inferior podemos ver los cambios que se subiran:

    • Una vez se suban los cambios la pestaña de pending changes ya no apareceran cambios pendientes:
