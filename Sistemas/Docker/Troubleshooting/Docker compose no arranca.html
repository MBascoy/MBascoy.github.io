Docker compose no arranca

Mbascoy.dev / Docker / Docker compose no arranca
Descripcion

A veces se da la situación de que el docker-compose no consigue arrancar los contenedores.

Algunos de los mensajes que se mostraban eran:

copy shim log error="read /proc/self/fd/14: file already closed" The last packet sent successfully to the server was 0 milliseconds ago. The driver has not received any packets from the server. [Warning] TIMESTAMP with implicit DEFAULT value is deprecated. Please use --explicit_defaults_for_timestamp server option (see documentation for more details). mysqld: Can't change permissions of the file 'ca-key.pem' (Errcode: 1 - Operation not permitted)
Comandos

En mi situación la solución era muy simple, el error se producia al intentar lanzar el docker-compose desde el sistema de archivos de windows, concretamente desde:

/mnt/c/Users/manuel.bascoy/lab/skillmatrix

Moviendo los archivos a la carpeta /home/usuario/skillmatrix solucionaba el problema y el docker-compose ya arrancaba sin fallos
