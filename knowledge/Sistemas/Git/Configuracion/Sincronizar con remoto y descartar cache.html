Sincronizar con remoto y descartar cache

Descripcion

Cuando hacemos un fetch y consultamos la ramas, nos aparecen ramas que ya han sido borradas en el repositorio remoto.

Metodo Linux

Esta situación se produce porque hemos descargado la información de esas ramas previamente y permanecen en la cache de nuestro loca, para descartar esa información podemos hacerlo con el siguiente comando:

git fetch --prune origin

De esta manera la información local de nuestro git mostrará las mismas ramas que el repositorio remoto
