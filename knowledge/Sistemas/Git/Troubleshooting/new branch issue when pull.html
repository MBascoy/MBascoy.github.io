new branch issue when pull

Descripcion

Cuando hacemos un pull en un repositorio nos aparece siempre un mensaje indicando que hay una nueva rama:

$ git pull
From https://github.com/SWO-GSDC-Spain/Academy-Team-02
 * [new branch]      st_6_getLanguages -> origin/st_6_getLanguages
Already up to date.
Explicacion

Esto se produce porque hay dos ramas con el mismo nombre pero cambiando solo algunas letras en Uppercase por Lowercase o viceversa

En nuestro caso tenemos una rama st_6_getLanguages y otra rama st_6_GetLanguages lo unico que cambia entre las dos ramas es la letra "G" que en una esta en mayuscula y la otra en minuscula

Esto podemos verlo si usamos el comando git remote show origin:

> git remote show origin
Solucion

Una manera de solucionar este fallo es eliminar una de las dos ramas con el nombre duplicado en el repositorio remoto y a continuacion ejecutar el siguiente comando:

> git fetch --prune origin
