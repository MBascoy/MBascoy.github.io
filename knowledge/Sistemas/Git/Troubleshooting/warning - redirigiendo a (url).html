warning - redirigiendo a (url)

Descripcion

Cuando hacemos un pull o un push obtenemos el siguiente warning:.

warning: redirigiendo a https://github.com/MBascoy/MBascoy.github.io/
Solucion

En este caso el warning se da porque la direccion configurada es una direccion http y no https por lo tanto el servidor nos redirige, podemos comprobarlo con el comando git remote-v

jericho@localhost:~/MBascoy.github.io> git remote -v
origin  http://github.com/MBascoy/MBascoy.github.io (fetch)
origin  http://github.com/MBascoy/MBascoy.github.io (push)

Para solucionar esto simplemente borramos la direccion asignada a la rama remota con el siguiente comando:

remote rm origin

Ahora a√±adimos la direccion correcta para que asi ya no nos redirija:

git remote add origin https://github.com/MBascoy/MBascoy.github.io/

y al hacer push (la primera vez) asociamos la rama main con la rama remota origin:

git push --set-upstream origin main

Ahora al hacer push o pull ya no obtenemos ese warning:
