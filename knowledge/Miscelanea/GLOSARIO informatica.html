Descripcion
  • CI/CD: Continuous Integration / Continuous Deployment (o Delivery)
  • CORS: Cross-Origin Resource Sharing
  • POJO: Plain Old Java Object
  • JPA: Java Persistence API
  • boilerplate: Código repetitivo que carece de lógica, como los setters y getters
  • Case
    • PascalCase: Cada palabra empieza con una mayuscula. VariableEjemplo
    • camelCase: Primera palabra con minuscula, el resto con mayuscula.variableEjemplo
    • kebab-case: Palabras todas en minuscula y separadas por "-".variable-ejemplo
    • snake_case: Palabras todas en minuscula y separadas por "_". variable_ejemplo
  • HTTP Query Parameters/Path Variables
    • Query Parameters: Al final de la url seguido del simbolo ? e.g: http://example.com/data?nombre=pepe nombre es el query parameter y pepe es su valor.
    • Path Variables: Es un valor dentro de la propia url que a simple vista no se distingue, suele colocarse al final. e.g: http://example.com/data/pepe pepe seria el valor del path variable, es algo que se gestiona internamente, no es tan facil de ver como las query parameters.
